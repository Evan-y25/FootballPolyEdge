# FootballPolyEdge — 波胆矩阵推导与对比方案

> 目标：以 Polymarket 的 **1X2（胜平负）** 盘口为输入，通过进球模型反解出一张**“真实”比分概率矩阵（波胆矩阵）**，再把它与 Polymarket 自己的**波胆（Exact Score）盘口**逐格对比，找出被高估/低估的比分（value edge）并可视化。
>
> 关系说明：本方案是 `DESIGN.md` §6.3「跨市场一致性」的**反向**用法。
> - §6.3：把 17 个波胆**聚合回** 1X2，做无风险套利校验。
> - 本方案：把 1X2 通过模型**展开成**完整比分矩阵，做基于模型的相对价值对比。
>
> ⚠️ 与 §6 的无风险套利不同：本方案的 edge **依赖模型假设是否正确**，属于「模型价值（value）」而非「无风险套利（arbitrage）」。两者在前端要明确区分。

---

## 1. 为什么能从 1X2 推出比分矩阵

足球进球数近似服从泊松分布。若给定主队期望进球 `λH`、客队期望进球 `λA`，则比分 `(i, j)`（主 i 球、客 j 球）的概率有解析式。而 1X2 概率正是比分矩阵的三块聚合：

- 主胜 `P_H = Σ_{i>j} P(i,j)`
- 平局 `P_D = Σ_{i=j} P(i,j)`
- 客胜 `P_A = Σ_{i<j} P(i,j)`

`P_H + P_D + P_A = 1`，因此 1X2 提供 **2 个独立约束**。独立泊松模型恰好有 **2 个自由参数 `(λH, λA)`** → 可唯一反解。解出 `(λH, λA)` 后即可生成整张矩阵。

> 关键认识：1X2 只能约束 2 个自由度。任何「比 2 个参数更复杂」的模型（双变量泊松的协方差、Dixon-Coles 的低分修正）都**无法**仅凭 1X2 识别其额外参数，必须由先验或其它市场（如大小球 O/U）补足。这也是为什么**矩阵内部（尤其平局结构）的偏差正是 edge 的来源，同时也是模型风险的来源**。

---

## 2. 总体流程（Pipeline）

```
Polymarket 1X2 盘口 (3 个二元市场)
        │  ① 取 mid 价
        ▼
   去抽水 / 归一化 (de-vig)  →  P_H*, P_D*, P_A*   (Σ=1 的“公平”1X2 概率)
        │
        ▼
   ② 拟合进球模型：求 (λH, λA[, ρ/λ3])  使模型 1X2 = P_H*,P_D*,P_A*
        │
        ▼
   ③ 生成比分矩阵 P_model(i,j)，i,j = 0..K
        │
        ▼
   ④ 聚合到 Polymarket 的 17 个比分桶（16 个具体比分 + "Other"）
        │
        ▼                                   Polymarket 波胆盘口 (17 个二元市场)
   ⑤ 逐桶对比：P_model  vs  P_market  ───────────────┘  (同样 de-vig → P_market*)
        │
        ▼
   ⑥ 输出：模型矩阵 / 市场矩阵 / 差值矩阵 + value edge 排行
```

## 3. 输入数据（复用现有 `app/gamma.py` / `app/state.py`）

每场 `Game` 已含：
- `home_win / draw / away_win`：1X2 三腿（`Outcome`，带 `yes_token`）。
- `scores: List[Outcome]`：波胆比分，`label` 形如 `"1 - 0"`、`"Other"`（见 `_score_label`）。

实时盘口由 `AppState._quote()` 给出 `{bid, ask, bid_size, ask_size, live}`。

**取价口径**：
- 用于**模型输入与“公平价”对比** → 取 mid = `(bid + ask) / 2`。
- 用于**可执行 edge**（要真实买/卖） → 买用 `ask`、卖用 `bid`。

### 3.1 Polymarket 波胆的 17 个桶

World Cup Exact Score 列出的具体比分均在 **0–3 球**范围内，超出部分归入 "Any Other Score"：

| 类别 | 比分 | 数量 |
|------|------|------|
| 主胜 | 1-0, 2-0, 2-1, 3-0, 3-1, 3-2 | 6 |
| 平局 | 0-0, 1-1, 2-2, 3-3 | 4 |
| 客胜 | 0-1, 0-2, 1-2, 0-3, 1-3, 2-3 | 6 |
| 其它 | Any Other Score（"Other"） | 1 |
| **合计** | | **17** |

## 4. 第①步：去抽水（de-vig）

三个二元市场的 mid 价 `q_H, q_D, q_A` 之和通常 `> 1`（抽水/overround）。归一化为公平概率：

- **比例法（baseline，推荐默认）**：`p_k = q_k / (q_H + q_D + q_A)`。
- **Shin / 幂律法（可选）**：修正「热门-冷门偏差」（favourite-longshot bias），对极端赔率更准；作为可配置项。

同法对波胆 17 桶去抽水得到 `P_market*(s)`（17 个价之和也会 >1，按比例归一）。

> 实现：在 `state.py` 增加 `_devig(quotes) -> probs`，1X2 与波胆共用。

---

## 5. 第②③步：拟合进球模型并生成矩阵

### 5.1 Baseline：独立泊松

```
P(i,j) = pois(i; λH) · pois(j; λA)
       = e^{-λH} λH^i / i!  ·  e^{-λA} λA^j / j!
```

**求解**：找 `(λH, λA)` 使
```
f1(λH,λA) = P_H(λH,λA) - P_H*  = 0
f2(λH,λA) = P_A(λH,λA) - P_A*  = 0      (P_D 自动满足，因为三者和为 1)
```
2 方程 2 未知，数值解：
- 截断到 `i, j ∈ [0, K]`（`K=10` 足够，尾部概率 < 1e-4）。
- 用 `scipy.optimize.fsolve` 或自实现的 2D Newton / `brentq` 嵌套；无 scipy 时用纯 Python 网格+局部细化。
- **初值**：`μ = λH + λA`（总进球，先验取世界杯均值 ~2.6），`supremacy = λH − λA` 由 1X2 倾向初始化；再迭代收敛。
- 解的存在性：1X2 几乎总能被独立泊松匹配（`(λH,λA)` 与 `(P_H,P_A)` 近似单调对应）；极端比分（如 P_D 异常高）可能需放宽到下一级模型。

### 5.2 增强：Dixon-Coles 低分修正

独立泊松会**系统性低估 0-0/1-1**、并轻微错配 1-0/0-1。Dixon-Coles 加入修正因子 `τ(i,j; λH, λA, ρ)`，仅改动 `(0,0),(0,1),(1,0),(1,1)` 四格：

```
P_DC(i,j) = τ(i,j) · pois(i;λH) · pois(j;λA)
```
`ρ` 为低分相关参数。**因 1X2 仅 2 约束，`ρ` 不可由单场识别** → 用**全局校准值**（对历史已结算比赛或当前所有场次的市场波胆做一次拟合得到稳定 `ρ`），再逐场只解 `(λH, λA)`。

### 5.3 增强：双变量泊松（需第 3 个市场）

若该场还有**大小球 / 总进球（O/U）** 市场，可引入 `Home=Y1+Y3, Away=Y2+Y3`，协方差 `=λ3`，用 O/U 提供的第 3 个约束（总进球分布）求解 `(λ1, λ2, λ3)`，更好地刻画进球相关性。无 O/U 市场时退回 5.1/5.2。

> 模型选择建议：**默认 5.2（DC + 全局 ρ）**；缺历史校准数据时退回 5.1；有 O/U 时可选 5.3。模型类型作为配置项 `SCORE_MODEL ∈ {poisson, dixon_coles, bivariate}`。

### 5.4 第④步：聚合到 17 桶

对 `i,j ∈ [0,K]` 的全矩阵：
- 凡 `(i,j)` 落在 Polymarket 列出的具体比分 → 累加到对应桶。
- 其余全部累加到 **"Other"**。

得到 `P_model*(s)`，且 `Σ_s P_model*(s) = 1`（构造保证）。

## 6. 第⑤⑥步：对比与 edge

### 6.1 三张矩阵 + 一张差值矩阵

按「主队进球（行 0..3+）× 客队进球（列 0..3+）」排布，外加 "Other" 单元：

| 矩阵 | 含义 |
|------|------|
| **模型矩阵** | `P_model*(i,j)` |
| **市场矩阵** | `P_market*(i,j)`（波胆 de-vig 后） |
| **差值矩阵** | `Δ(i,j) = P_model* − P_market*` |

- `Δ > 0`：模型认为该比分概率高于市场标价 → 市场**低估** → 倾向**买入** YES。
- `Δ < 0`：市场**高估** → 倾向**卖出** YES。

### 6.2 Value edge（可执行口径）

公平概率用模型 `f_s = P_model*(s)`；执行价用真实盘口（`ask_s` 买、`bid_s` 卖，**未去抽水**，因为真实成交付的是含抽水的价）：

- **买入价值**：`edge_buy(s) = f_s − ask_s`，`> 阈值 τ_edge` 时标记「买 YES」。
- **卖出价值**：`edge_sell(s) = bid_s − f_s`，`> 阈值` 时标记「卖 YES」。
- 期望收益按单注赔付 1 计；可成交量取该桶盘口 `ask_size / bid_size`。

> 阈值 `τ_edge`（如 0.02）用于过滤噪声与模型误差；可配置。

### 6.3 一致性校验（必做，防自欺）

1. **归一校验**：模型与市场矩阵各自 `Σ ≈ 1`（市场未必，记录其 overround）。
2. **1X2 回算**：把**模型矩阵**聚合回 1X2，必须等于输入 `P_H*,P_D*,P_A*`（构造保证，作单测）。
3. **Other 桶对账**：模型尾部 `P_model*("Other")` 与市场 `P_market*("Other")` 差异是重要信号——若模型尾部明显小于市场，说明市场把概率压在长尾上（或反之）。
4. **跨市场闭环**：把**市场波胆**聚合回 1X2，与市场 1X2 的差异即 §6.3 的跨市场不一致；本方案的模型矩阵应落在「市场 1X2」与「市场波胆聚合」之间，可三者并列展示。

## 8. 验证与校准

1. **单元测试**：构造已知 `(λH, λA)` → 生成 1X2 → `fit_lambdas` 应还原原 λ（容差内）；矩阵 `Σ=1`；1X2 回算一致。
2. **全局 `ρ` 校准脚本**：对当前所有场次（或历史已结算场次）最小化 `Σ (P_model* − P_market*)²` 拟合一个稳定 `ρ`，定期更新。
3. **回测**：对已结算比赛，比较「模型矩阵 / 市场矩阵」对真实比分的对数损失（log-loss）/ Brier 分数，判断模型是否真比市场更准（决定 value edge 是否可信）。
4. **数值稳健性**：极端 1X2（某腿 → 0/1）时的解的稳定性与退化处理。

---

## 9. 风险与局限

1. **模型风险**：edge 完全依赖泊松/DC 假设。1X2 只约束 2 个自由度，矩阵内部结构（尤其平局分布）是**模型外推**，不是市场事实 → value edge **不是无风险套利**。
2. **识别不足**：单场无法识别相关性/低分修正参数，须靠全局校准或 O/U 市场补足；校准漂移会直接影响 edge。
3. **去抽水方法敏感**：比例法 vs Shin 会改变 `P_*`，进而改变 λ 与矩阵；需作为可比较的配置项。
4. **"Other" 桶**：模型尾部与市场长尾的口径差异需专门对账，避免把口径差当成 edge。
5. **流动性/执行**：value edge 需用 `ask/bid` 与 `size` 评估真实可成交性；未计手续费、gas、滑点。
6. **盘口质量**：单边/无盘口时回退到 Gamma 初始价（见 `_quote` 兜底），此时 edge 仅供参考。
